Winnow Design System

Design systems are Pattern Libraries: a big box of UI Lego pieces that can be assembled in any ways. All the pieces may be consistent, but that doesn't mean the assembled results will be. It has structure and meaning to every combination.

Examples here are meant for communicating responsive designs to Winnow development teams written with Jekyll, SASS and Vanilla JavaScript. You can recreate components according to the framework (e.g. React) used for your application.

  • Widgets - Standard layouts for presenting different types of content
  • Flows - Sequence of screens that users will interact for specific cases
  • Components - Group of elements to fulfil a certain function
  • Elements - Basic building blocks of everything

Responsiveness

Winnow elements is optimized for vertical reading, so by default on mobile (up to 768px):

  • Columns are stacked vertically
  • Navbar will be hidden

Concept for Winnow Dashboard

Winnow Dashboard is made up of tile elements to build 2-dimensional Metro-like, Pinterest-like or whatever-you-like grids. It is responsive and the tiles will stack above one another when displayed in mobile. You can resize this browser to see it live.

Each tile below represent a widget where different layouts are available for mix-and-match.
